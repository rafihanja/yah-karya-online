# YouTube Summarizer

> **One-liner:** Guidelines for extracting YouTube video transcripts using `youtube-transcript-api` in Python and generating detailed, structured summaries using the STAR + R-I-S-E analysis framework.

## When to Use

- When summarizing educational videos, lectures, tutorials, or conference talks from a YouTube link.
- When extracting raw timestamps and text transcripts from videos for analysis.
- When creating structured documentation capture guides for video content.

## Why This Exists

Reading long transcripts is time-consuming, while short summaries generated by standard LLMs often miss key technical points, data references, and developer steps. Additionally, attempting to download transcripts by scraping frontends directly fails when Google introduces layout changes or updates antibot blocks. Standardizing on the official `youtube-transcript-api` library and applying the STAR + R-I-S-E summary model ensures complete and accurate documentation.

## ALWAYS DO THIS

- **Verify `youtube-transcript-api` availability first** — Check for the library in Python using `python3 -c "import youtube_transcript_api"` on boot.
- **Implement language fallbacks** — Request transcripts with language order preferences (e.g. `languages=['en', 'id']`) to handle auto-generated tracks.
- **Extract video IDs via robust regex** — Support multiple URL schemas including `youtube.com/watch?v=`, `youtu.be/`, and mobile paths.
- **Apply the STAR + R-I-S-E framework** — Structure the output with a Situation/Task/Action/Result executive synthesis, followed by section breakdowns, key insights, and terms.
- **Display progress steps** — Show visual status guides (e.g., `[████░░░░] 50% - Step 3/5: Extracting Transcript`) during slow execution runs.

## NEVER DO THIS

- ❌ **DO NOT** scrape transcripts using direct HTML page scraping or string regex on main pages. **Why fails:** Google's frontend formats change frequently, which breaks scraper rules and gets IP addresses blocked. **Instead:** Fetch transcripts using `youtube-transcript-api` or the YouTube Data API.
- ❌ **DO NOT** generate short, generic summaries (e.g., "This video explains machine learning in 10 minutes"). **Why fails:** Fails to capture technical details, commands, formulas, or code structures. **Instead:** Write detailed section-by-section summaries.
- ❌ **DO NOT** access transcript arrays without catching exceptions like `TranscriptsDisabled` or `NoTranscriptFound`. **Why fails:** Causes the execution script to crash when a video lacks subtitles. **Instead:** Handle exceptions and inform the user.
- ❌ **DO NOT** store temporary transcript files in global root folders without checking writing permissions. **Why fails:** Raises write access errors in read-only directories. **Instead:** Write files to the workspace or system temporary folder (`/tmp` or OS equivalents).

## Summarization Framework: STAR + R-I-S-E

Format summaries using the structured framework:
- **STAR:** Situation, Task, Action, Result (for executive summary).
- **R-I-S-E:** Resources, Insights, Steps, Essentials (for detailed sections).

## Examples

### ✅ Good — Python-Based Transcript Extractor with Language Fallbacks

```python
import sys
import re
from youtube_transcript_api import YouTubeTranscriptApi, TranscriptsDisabled, NoTranscriptFound

def extract_video_id(url: str) -> str:
    # 1. Regex handles both standard and shortened YouTube URLs
    pattern = r"(?:v=|\/watch\?v=|\/\?v=|\/embed\/|\/youtu\.be\/|\/v\/|watch\?v%3D|watch\?v=)([^#\&\?]*)"
    match = re.search(pattern, url)
    if match and len(match.group(1)) == 11:
        return match.group(1)
    raise ValueError("Invalid YouTube URL format")

def fetch_transcript(url: str):
    try:
        video_id = extract_video_id(url)
        print(f"📹 Processing Video ID: {video_id}...")

        # 2. Request transcript with fallbacks
        transcript = YouTubeTranscriptApi.get_transcript(video_id, languages=["en", "id"])
        
        # 3. Join text components safely
        full_text = " ".join([segment["text"] for segment in transcript])
        print(f"✅ Success! Transcript length: {len(full_text)} characters")
        
        return full_text
    except TranscriptsDisabled:
        print("❌ Transcripts are disabled for this video.")
        sys.exit(1)
    except NoTranscriptFound:
        print("❌ No matching language transcript found.")
        sys.exit(1)
    except Exception as e:
        print(f"❌ Error: {e}")
        sys.exit(1)

if __name__ == "__main__":
    if len(sys.argv) < 2:
        print("Usage: python transcript_fetcher.py <youtube_url>")
        sys.exit(1)
    fetch_transcript(sys.argv[1])
```

Why this passes: Uses regex to extract video IDs, implements language fallbacks, catches specific library exceptions, and joins text segments safely.

### ❌ Bad — HTML Scraper and Missing Error Checks

```python
import sys
import urllib.request
import re

# ERROR 1: Trying to extract transcripts by scraping the main HTML page
def scrape_transcript_unsafe(url):
    # ERROR 2: No URL format validations
    html = urllib.request.urlopen(url).read().decode('utf-8')
    
    # ERROR 3: Brittle regex trying to find ytInitialPlayerResponse variables
    matches = re.findall(r'"captionTracks":\[(.*?)\]', html)
    
    // ERROR 4: Lacks try/catch blocks - raises IndexError if no subtitles exist
    caption_url = re.search(r'"baseUrl":"(.*?)"', matches[0]).group(1)
    
    # Scrapes raw XML
    xml = urllib.request.urlopen(caption_url).read().decode('utf-8')
    print("Downloaded raw transcript successfully")
    return xml
```

Why this fails: Scrapes HTML page structures, lacks URL validation checks, uses fragile regex, and lacks error wrappers.

## Failure Modes

- **Subtitles Disabled Crash:** Failing to handle videos that have transcripts disabled, which crashes the execution thread.
- **The Empty Translation Return:** Querying a language code that is not supported by the video, returning an empty array.
- **The Quota Lockout:** Scraping thousands of characters inside loops, causing Google to block the IP address.
